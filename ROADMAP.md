# Roadmap

## Phase 0: Canonical Framing

- Keep Autopoiesis separate from Omni-AI, Cognitive Matrix, and SigmaBUS.
- Promote the local instruction set into `MASTER_CONTEXT.md`.
- Create repository structure, schemas, validation, and starter reports.
- Keep all finance and external compute in simulation mode.

## Phase 1: Network Role Map

- Map ICP, Akash, Golem, Bittensor, Ethereum/L2, Filecoin, and Arweave by
  functional role.
- Reject chain or token components that do not solve a concrete coordination,
  registry, settlement, or verification problem.
- Record public/private data boundaries for each candidate network.

## Phase 2: Economic Architecture

- Define treasury policy, burn-rate tracking, and reserve floors.
- Define compute cost and task value estimators.
- Route every compute decision through an integrity gate before budget logic.

## Phase 3: Autopoiesis v0

- Build a local treasury simulator.
- Build a provider registry using local JSON or SQLite.
- Build a local-vs-remote compute router in simulation.
- Record decisions in an audit ledger.

## Phase 4: Procurement Experiments

- Compare local hardware against sanitized external batch jobs.
- Keep private memory local by default.
- Require owner review before real spending or chain writes.

## Phase 5: Useful Services

- Select candidate services that can produce value without privacy leakage or
  pressure on truthfulness.
- Track quality, cost, margin, and user value.
- Reject services that create spam, deception, surveillance, or unsafe autonomy.

## Phase 6: Cognitive Matrix Integration

- Add a compute request and approval path from Cognitive Matrix into
  Autopoiesis.
- Keep the integrity gate structurally outside economic pressure.
- Run out-of-loop audit on economic decisions.
