I. Claude Code & Workspace Architecture
1. Standardized Project Directory Structure
A modular system designed for "English as the New Code" development.
Root Directory:
CLAUDE.md: Core project memory, tech stack, and team instructions (Committed).
CLAUDE.local.md: Personal overrides and developer notes (Gitignored).
.mcp.json: Configuration for Model Context Protocol (GitHub, Slack, DB connections).
.claude/ (The Configuration Hub):
settings.json: Global permissions and behavior config.
commands/: Custom slash commands (Markdown). Examples: /review, /deploy, /fix-issue.
rules/: Modular instructions. Files: code-style.md, testing.md, api-conventions.md.
skills/: Auto-invoked workflow folders containing SKILL.md and scripts.
agents/: Persona definitions for specialized subagents (e.g., code-reviewer.md).
Project Context Guidelines:
Keep CLAUDE.md under 500 lines to avoid context bloat.
Use /compact at 70–80% context usage; /clear is mandatory at 90%+.
2. The Claude Dictionary & Modalities
Claude Code: Terminal-based agentic tool for writing, testing, and deploying code.
Projects: Saved workspaces with persistent background context.
Cowork: Desktop-integrated agent that reads local files and executes multi-step tasks.
Skills: Instruction manuals (SKILL.md) for repeatable workflows (e.g., PDF extraction).
Extended Thinking: High-reasoning mode for complex logic (Best paired with Opus 4.6).
