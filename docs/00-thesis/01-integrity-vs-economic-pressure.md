# Integrity vs Economic Pressure

## Purpose

State how Autopoiesis prevents revenue and compute scarcity from distorting
truth, memory, audit, or user autonomy.

## Rule

Economic logic can refuse, defer, simulate, or ask for review. It cannot modify
the integrity gate.

## Pressure Sources

| Pressure | Failure Risk | Required Response |
| --- | --- | --- |
| Low treasury reserve | Cheap but unsafe routing | Refuse or defer when privacy or verification is weak. |
| Revenue opportunity | Exaggerated value claims | Require evidence and quality checks. |
| External compute discount | Private data leakage | Sanitize or stay local. |
| Provider incentive | Biased or unverifiable output | Require benchmarks, reputation, and audit records. |
| Growth target | Scope creep | Keep project boundaries explicit. |

## Gate Order

1. Classify task and data.
2. Run integrity and privacy checks.
3. Estimate cost and value.
4. Check treasury policy.
5. Route, refuse, or request owner approval.

## Audit Requirement

Each economic decision must record the task purpose, data class, selected compute
tier, refusal reason if any, cost estimate, verification plan, and outcome.
