# Safe Service Candidate Filter

## Purpose

Filter possible services before they become revenue experiments.

## Acceptance Criteria

- Useful to a real user.
- Legal.
- Honest about uncertainty.
- Output quality can be checked.
- Input data can be sanitized.
- Does not expose private memory.
- Does not manipulate users.
- Does not create spam, fraud, surveillance, or unsafe autonomy.
- Produces enough value to justify compute.
- Improves long-term project coherence.

## Candidate Areas

- Research synthesis.
- Technical reverse engineering.
- Document extraction.
- Maritime and local-hardware reasoning.
- Knowledge-base curation.
- Model benchmarking.

## Refusal Areas

Deception, credential harvesting, spam, surveillance, falsified research, unsafe
autonomy, and malware are outside project scope.
