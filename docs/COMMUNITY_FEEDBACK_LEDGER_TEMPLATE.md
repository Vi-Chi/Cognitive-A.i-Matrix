# COMMUNITY_FEEDBACK_LEDGER.md

Use this ledger to preserve useful public feedback without letting popularity override architecture.

## Feedback Entry Template

```markdown
## YYYY-MM-DD — <channel/thread>

- Source/channel:
- Link/reference:
- Feedback type: bug / idea / criticism / question / collaboration / safety / docs / benchmark / hardware / license / moderation
- Summary:
- Evidence quality: high / medium / low / unknown
- Relevant module/docs:
- Requires response: yes/no
- Safe response draft:
- Requires project action: yes/no
- Suggested issue/task:
- Canon impact: none / possible / likely / blocked
- Status: new / triaged / accepted / rejected / implemented / closed
```
