# Network Role Map

## Purpose

Assign candidate networks to concrete roles and reject unsupported hype.

| System | Candidate Role | Phase 0 Decision |
| --- | --- | --- |
| ICP | Coordination, registry, public state, canister concept study. | Research and simulate first. |
| Akash | Rented decentralized compute for sanitized batch jobs. | Research pricing and deployment boundaries. |
| Golem | Requestor/provider task execution experiments. | Research bounded task model. |
| Bittensor | Useful-output market research. | Study incentives, do not depend on it. |
| Ethereum/L2 | Optional settlement, escrow, ownership, or attestations. | Out of v0 unless a concrete role appears. |
| Filecoin/Arweave | Public durable storage. | Public artifacts only. |
| Local hardware | Sovereign baseline compute. | Default tier. |

## Boundary

Private memory, private logs, credentials, local endpoints, and raw vessel data
remain local unless the owner approves a specific export.
