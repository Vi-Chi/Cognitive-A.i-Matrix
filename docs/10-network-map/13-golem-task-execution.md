# Golem Task Execution

## Candidate Role

Golem may fit bounded task execution where inputs are public, sanitized, or
synthetic and outputs can be verified.

## Fit

- Batch transforms.
- Benchmark jobs.
- Synthetic simulations.
- Public research corpus processing.

## Poor Fit

- Private memory.
- Stateful agent control.
- Long-running services.
- Jobs with unverifiable output quality.

## Next Research

Document provider trust, sandbox behavior, input privacy, result verification,
and pricing.
